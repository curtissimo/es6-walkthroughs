// In the days before Unicode, we had 2^8 = 256 // characters and, then, 2^16 = 65,536 characters // in our character sets. We could represent each // of those characters as a number, or code, for // the specific character encoding we used in // our software. Pre-Unicode ECMAscript provides // the String.fromCharCode method to build strings // from those character codes. // This converts the code 72 to "H" and 105 to "i" // and concatenates them to form "Hi". console.log(String.fromCharCode(72, 105)); // But, because we had a limit of 65,536 // characters, the String.fromCharCode method // would modulo the argument to prevent those // pesky exceptions that early Web "programmers" // hated. console.log(String.fromCharCode(72 + 65536, 105)); // Then, Unicode came along with its // Supplementary Multilingual Plane and its // Supplementary Ideographic Plane and more, // characters represented by more than one // character code which Unicode calls a "code // point". Now, String.fromCharCode doesn't work. console.log(String.fromCharCode(127776)); // String.fromCodePoint exists to resolve the // values outside the Basic Multilingual Plane. console.log(String.fromCodePoint(127776));
