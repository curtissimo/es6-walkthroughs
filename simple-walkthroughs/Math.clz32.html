// With the addition of binary value handling in // ECMAscript, as well as applications such as // WebGL, more sophisticated bit operations need // to exist in ECMAscript. One of these is the // Count Leading Zeros (32-bit representation). // It has applications in converting integers to // floating-point numbers, fast data compression, // and managing overflows due to multiplication. // For 0, which we would represent as 32 zeroes // in a 32-bit representation, will return 32 // for the CLZ operation. console.log(Math.clz32(0)); // For 1, which would have the least significant // bit set to 1 and all others 0, will return 31. console.log(Math.clz32(1)); // And so on for powers of 2, with each in the // series reducing the number of leading zeroes // by one. console.log(Math.clz32(2)); console.log(Math.clz32(4)); console.log(Math.clz32(8)); console.log(Math.clz32(16)); console.log(Math.clz32(32)); // Negative numbers are converted to unsigned // integers, first. So, -1 is the same as the // maximum unsigned integer and, therefore, has // no leading zeroes. console.log(Math.clz32(-1)); // The conversion of a floating-point number to // an unsigned integer involves truncating its // mantissa instead of rounding. console.log(Math.clz32(1)); console.log(Math.clz32(1.999999));
