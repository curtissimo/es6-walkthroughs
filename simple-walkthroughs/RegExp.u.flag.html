// I'm not going to get all into Unicode because // I feel pretty certain that you would read for // hours and hours and hours and hours and ... // Suffice it to say, some Unicode characters // exist outside the Basic Multilingual Plane // and exist outside of the first 65,536 code // points. Because of that, we have to // specify two code points to make the character // like you can see in this string. var s = 'I got the \uD83D\uDCA9 on me!'; // Regular expressions through ECMAscript 5 fail // on understanding that one character could have // multiple code points. Because of that, the // following regular expression fails because the // dot operator, while supposedly matching a // single character, actually matches a single // code point. console.log(s.match(/I got the . on me!/)); // Now, with the "u" flag, regular expressions // with character patters like ".", "?", and // "{}" will correctly recognize multi-code- // point characters. console.log(s.match(/I got the . on me!/u));
