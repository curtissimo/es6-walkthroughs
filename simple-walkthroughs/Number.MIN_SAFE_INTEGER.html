// Previous to ECMAscript 6, we had no easy way to // know the lower bounds of mathematical // calculations. Instead, you had to know the IEEE // specification that governs the representation // of the numbers and how that standard defined // all of the mathematical operations. With ES6, // we no longer have to know all that because it // provides a constant for us to know the lowest // integer from which we can subtract 1 and get an // accurate calculation: Number.MIN_SAFE_INTEGER // With that in mind, we can confidently have // ECMAscript calculate the following sum. console.log(Number.MIN_SAFE_INTEGER - 1); // However, ECMAscript specifically informs us that // the following calculations will yield results // that we do not expect. console.log(Number.MIN_SAFE_INTEGER - 2); console.log(Number.MIN_SAFE_INTEGER - 3); console.log(Number.MIN_SAFE_INTEGER - 4); console.log(Number.MIN_SAFE_INTEGER - 5); console.log(Number.MIN_SAFE_INTEGER - 6); // On my computer, MIN_SAFE_INTEGER - 1 and // MIN_SAFE_INTEGER - 2 yield the same results!
